This paper was published at FormaliSE 2025, the IEEE/ACM International Conference on Formal Methods in Software Engineering, held in Ottawa, Canada.

The work studies mutation operators for Alloy — a relational modeling language used in formal software specification. Mutation testing in formal languages requires mutation operators that reflect the kinds of mistakes users actually make, rather than arbitrary syntactic perturbations. We refined the existing set of Alloy mutation operators based on an empirical analysis of common user errors, making mutation testing more representative of real-world faults.

I contributed to the implementation and evaluation components of this work, collaborating with Aleksandar Jovanovic and our advisor Dr. Allison Sullivan.

This was my first peer-reviewed publication in the formal methods space.
